Este código refere-se ao **Auto Robô v2.0**, um sistema de automação desenvolvido em **Node.js** utilizando a biblioteca **Puppeteer**. O objetivo principal do projeto é automatizar tarefas repetitivas em sistemas corporativos internos da Vivo/GVT, especificamente os sistemas **GOI** (Gestão de Ordens de Instalação), **WFM** (Workforce Management) e **GPS**.

Abaixo detalho os principais aspectos do projeto contidos no arquivo:

### 1. Objetivo Funcional
O robô executa o seguinte fluxo de trabalho:
*   **Login Automático:** Acessa e realiza login nos sistemas GOI e WFM usando credenciais configuradas.
*   **Navegação:** Abre abas adicionais para os sistemas WFM e GPS.
*   **Injeção D0:** Navega até a aba de "Injeção D0" no sistema de agendamento, executa uma função JavaScript interna (`injecaoAnalitico`) para carregar dados e filtra atividades.
*   **Atribuição de Atividades:** Busca uma ordem de serviço específica (PON/Atividade) na tabela e clica no botão de atribuição.
*   **Diagnóstico:** Tira screenshots em caso de erro ou sucesso e gera logs coloridos detalhados.

### 2. Arquitetura Técnica
O projeto foi refatorado para ser **modular**, **escalável** e **fácil de manter**:
*   **ES Modules:** Utiliza `import/export` moderno do JavaScript.
*   **Separação de Responsabilidades:**
    *   `src/config/`: Centraliza configurações (URLs, credenciais, timeouts) via arquivo `.env`.
    *   `src/browser/`: Gerencia a inicialização do Chrome (pode conectar a um Chrome existente ou lançar um novo) e utilitários de página.
    *   `src/services/`: Contém a lógica de negócio (login, injeção, navegação).
    *   `src/utils/`: Funções auxiliares como logger colorido, espera inteligente com retry (`waitForSelectorWithRetry`), captura de screenshots e centralização de seletores CSS.
*   **Configuração Flexível (.env):** Permite alternar entre modos de execução sem alterar o código fonte.

### 3. Recursos Avançados Implementados
*   **Modo Híbrido de Navegador:**
    *   Pode conectar-se a uma instância do Chrome já aberta via Debug Port (útil para reutilizar sessões logadas ou contornar CAPTCHAs manualmente).
    *   Pode lançar uma nova instância do Chrome gerenciada pelo Puppeteer.
*   **Modo de Desenvolvimento/Teste (Mocking):**
    *   Como os sistemas reais só acessíveis via VPN corporativa, o código inclui um **Servidor Mock Local** (`scripts/mock-server.js`) que simula as respostas HTML dos sistemas GOI/WFM. Isso permite desenvolver e testar a lógica em casa sem estar conectado à rede da empresa.
*   **Tratamento de Erros Robusto:**
    *   Logs coloridos com níveis (info, warn, error, debug).
    *   Retry automático em falhas de seleção de elementos.
    *   Captura automática de screenshots em caso de falha crítica para diagnóstico posterior.
*   **Segurança:** Credenciais não ficam hardcodadas; são lidas do arquivo `.env` (que está no `.gitignore`).

Autistas muitas vezes brilham na detecção de padrões e inconsistências. Use isso a seu favor para criar um código que não quebra facilmente.

*   **Centralização Absoluta de Seletores:**
    *   *Por que:* Sites corporativos mudam IDs de classes frequentemente. Se o CSS mudar, você não quer caçar erros em 10 arquivos.
    *   *Ação:* Mantenha rigorosamente o arquivo `src/utils/selectors.js`. Comente cada seletor explicando *o que* ele é (ex: `// Botão de entrar após preencher senha`).

    *   *Por que:* Você mencionou gostar de logs coloridos. Isso ajuda muito no diagnóstico rápido sem precisar ler linhas infinitas de texto branco.

### 1. Crie o arquivo `src/cli/menu.js`

Este arquivo usa a biblioteca nativa `readline` do Node.js para criar um menu interativo.

```javascript
// src/cli/menu.js
import readline from 'readline';
import { execSync } from 'child_process';
import logger from '../utils/logger.js';
import { automationConfig, systemsConfig } from '../config/index.js';

const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout
});

const question = (query) => new Promise((resolve) => rl.question(query, resolve));

// Cores para o menu
const COLORS = {
  reset: '\x1b[0m',
  cyan: '\x1b[36m',
  green: '\x1b[32m',
  yellow: '\x1b[33m',
  red: '\x1b[31m',
  bright: '\x1b[1m'
};

const showHeader = () => {
  console.clear();
  console.log(`${COLORS.cyan}
╔════════════════════════════════════════════════════════╗`);
  console.log(`${COLORS.cyan}║  🤖 AUTO ROBO v2.0 - Menu Principal              ║`);
  console.log(`${COLORS.cyan}║  Desenvolvido por: xAgron                        ║`);
  console.log(`${COLORS.cyan}╚════════════════════════════════════════════════════════╝${COLORS.reset}
`);
};

const checkMockServer = async () => {
  try {
    // Tenta fazer um fetch rápido para localhost:3456
    const response = await fetch('http://localhost:3456/login', { method: 'HEAD' });
    if (response.ok) {
      logger.success('Mock Server está ONLINE ✅');
      return true;
    }
  } catch (e) {
    logger.error('Mock Server está OFFLINE ❌');
    logger.info('Execute "npm run mock" em outro terminal.');
    return false;
  }
};

const checkVPN = async () => {
  try {
    // Tenta resolver DNS do sistema corporativo
    const dns = await import('dns').then(m => m.promises);
    await dns.lookup('goi.vivo.com.br');
    logger.success('DNS Corporativo resolvido (VPN/Cabo provável) ✅');
    return true;
  } catch (e) {
    logger.error('DNS Corporativo NÃO resolvido (Sem VPN/Cabo) ❌');
    return false;
  }
};

export const showMainMenu = async () => {
  showHeader();
  
  console.log(`${COLORS.bright}Escolha o modo de execução:${COLORS.reset}`);
  console.log(`  ${COLORS.green}[1]${COLORS.reset} Modo Desenvolvimento (Mock Local)`);
  console.log(`  ${COLORS.cyan}[2]${COLORS.reset} Modo Produção (Sistemas Reais)`);
  console.log(`  ${COLORS.yellow}[3]${COLORS.reset} Diagnóstico de Rede`);
  console.log(`  ${COLORS.red}[0]${COLORS.reset} Sair`);
  console.log('');

  const choice = await question('👉 Digite sua opção: ');

  switch (choice.trim()) {
    case '1':
      await handleDevMode();
      break;
    case '2':
      await handleProdMode();
      break;
    case '3':
      await handleDiagnostics();
      break;
    case '0':
      logger.info('Saindo...');
      rl.close();
      process.exit(0);
      break;
    default:
      logger.warn('Opção inválida. Tente novamente.');
      await new Promise(r => setTimeout(r, 1500));
      return showMainMenu(); // Recursão para mostrar o menu de novo
  }
};

const handleDevMode = async () => {
  logger.info('Configurando Modo Desenvolvimento...');
  
  // Verifica se o mock está rodando
  const isMockUp = await checkMockServer();
  
  if (!isMockUp) {
    const startMock = await question('⚠️  Mock Server não detectado. Deseja iniciar agora? (s/n): ');
    if (startMock.toLowerCase() === 's') {
      logger.info('Iniciando Mock Server em background...');
      // Nota: Em Windows, spawn detached é complexo. 
      // O ideal é pedir para o usuário abrir outro terminal.
      logger.warn('Por favor, abra OUTRO terminal e execute: npm run mock');
      await question('Pressione ENTER quando o Mock Server estiver pronto...');
    } else {
      logger.warn('Cancelando execução. Inicie o Mock Server manualmente.');
      rl.close();
      process.exit(0);
    }
  }

  // Configura variáveis para este processo
  process.env.MODO_TESTE = 'true';
  process.env.LOGIN_URL = 'http://localhost:3456/login';
  process.env.AGENDAMENTO_URL = 'http://localhost:3456/agendamento/atendimento.php';
  
  logger.success('Modo Desenvolvimento Ativado! Iniciando robô...');
  await new Promise(r => setTimeout(r, 1000));
  rl.close();
  // Retorna controle para o index.js continuar
  return 'DEV';
};

const handleProdMode = async () => {
  logger.info('Configurando Modo Produção...');
  
  // Verifica VPN
  const hasVPN = await checkVPN();
  
  if (!hasVPN) {
    const force = await question('⚠️  VPN/Rede Corporativa não detectada. Deseja continuar mesmo assim? (s/n): ');
    if (force.toLowerCase() !== 's') {
      logger.warn('Conecte-se à VPN e tente novamente.');
      rl.close();
      process.exit(0);
    }
  }

  // Configura variáveis para produção
  process.env.MODO_TESTE = 'false';
  // As URLs reais já estão no .env, então não precisamos sobrescrever,
  // a menos que queira garantir. O config/systems.js lê do .env.
  
  logger.success('Modo Produção Ativado! Iniciando robô...');
  await new Promise(r => setTimeout(r, 1000));
  rl.close();
  return 'PROD';
};

const handleDiagnostics = async () => {
  logger.info('Executando diagnóstico...');
  console.log('');
  await checkMockServer();
  await checkVPN();
  console.log('');
  await question('Pressione ENTER para voltar ao menu...');
  return showMainMenu();
};
```

```javascript
// Função principal de execução da automação
const runAutomation = async () => {
  let browser = null;
  let page = null;
  try {
    logger.divider();
    
    // 1. Inicializa navegador
    logger.info('main', '📦 Inicializando navegador...');
    const { browser: br, page: pg } = await initBrowser();

    await loginService.loginGOI(page);
    await sleep(1000);
```

```javascript
import express from 'express';
import { createServer } from 'http';
import { Server } from 'socket.io';
import path from 'path';
import { fileURLToPath } from 'url';
import { runAutomation } from '../../index.js'; // Importa a lógica do robô

const app = express();
const httpServer = createServer(app);
const io = new Server(httpServer, {
  cors: { origin: "*" }
});

const PORT = process.env.WEB_PORT || 3000;

  res.sendFile(path.join(__dirname, '../../public/index.html'));

io.on('connection', (socket) => {
  console.log('🔌 Cliente conectado ao Dashboard');

  socket.emit('status', { running: isRunning });

  // Evento: Usuário clicou em "Iniciar Robô"
  socket.on('start-robot', async (data) => {
    if (isRunning) {
      socket.emit('log', { level: 'warn', msg: '⚠️ Robô já está em execução.' });
      return;
    }

    const mode = data.mode; // 'DEV' ou 'PROD'
    isRunning = true;
    socket.emit('status', { running: true });
    socket.emit('log', { level: 'info', msg: `🚀 Iniciando Robô no modo: ${mode}` });

    try {
      // Configura variáveis de ambiente baseado na escolha do usuário
      if (mode === 'DEV') {
        process.env.MODO_TESTE = 'true';
        process.env.LOGIN_URL = 'http://localhost:3456/login';
        process.env.AGENDAMENTO_URL = 'http://localhost:3456/agendamento/atendimento.php';
      } else {
        process.env.MODO_TESTE = 'false';
        // URLs reais vêm do .env
      }

      // Executa a automação
      await runAutomation(socket); // Passamos o socket para enviar logs
      
      socket.emit('log', { level: 'success', msg: '✅ Automação concluída!' });
    } catch (error) {
      socket.emit('log', { level: 'error', msg: `❌ Erro Crítico: ${error.message}` });
    } finally {
      isRunning = false;
      socket.emit('status', { running: false });
    }
  });

  // Evento: Usuário clicou em "Iniciar Mock Server" (opcional, pode ser externo)
  socket.on('start-mock', () => {
    socket.emit('log', { level: 'info', msg: '💡 Para iniciar o Mock Server, use o terminal: npm run mock' });
```

```javascript
        }
    }).catch(() => {}); 
    
    await sleep(1000);

    // 4. Abre abas adicionais

    // const extraTabs = await navigationService.openSystemTabs(browser, page);

      await injectionService.buscarEAtribuirAtividade(page, automationConfig.atividadeTeste);
    }

    // 7. Screenshot final

    await captureAndSave(page, 'conclusao');
```

### ✅ Solução: Forçar a Configuração no Momento da Execução

Em vez de depender apenas de `process.env`, vamos passar a configuração correta diretamente para os serviços ou recarregar a configuração dinamicamente.

#### 1. Atualize `src/server/index.js`

Vamos modificar a lógica para garantir que as URLs corretas sejam usadas. A maneira mais segura é **recarregar o módulo de configuração** ou **passar as URLs como parâmetros**. Como refatorar todos os serviços agora seria complexo, vamos usar um truque eficaz: **Recarregar o dotenv e forçar a atualização das variáveis ANTES de importar a lógica do robô.**

Mas há um problema: os `imports` no topo do arquivo são executados antes do código rodar.

**A melhor estratégia para o seu caso atual:**
Vamos criar uma função que **reinicia o processo** com as variáveis de ambiente corretas, ou ajustar o `src/config/systems.js` para ler as variáveis **dinamicamente** a cada chamada, não apenas na inicialização do módulo.

Vamos optar pela solução mais simples e eficaz agora: **Modificar `src/config/systems.js` para ser dinâmico.**

#### Passo 1: Edite `src/config/systems.js`

Altere o arquivo para que ele leia `process.env` **dentro das funções/objetos** ou use uma função getter, mas como estamos exportando um objeto padrão, o ideal é garantir que o `dotenv` seja recarregado ou que as variáveis estejam corretas **antes** da importação.

Como o `index.js` do servidor importa `runAutomation` do `index.js` raiz, e este importa os serviços, a configuração já está "travada".

**Solução Rápida (Hardcode Inteligente no Server):**

No `src/server/index.js`, vamos interceptar a execução e definir as variáveis **globalmente** e depois exigir que o código as use. Mas o problema é o cache.

Vamos fazer o seguinte: **Reiniciar o processo filho com as vars corretas.** É a forma mais limpa sem refatorar toda a arquitetura de config agora.

Substitua o conteúdo de `src/server/index.js` por este, que usa `child_process` para rodar o robô com as envs corretas:

```javascript
// src/server/index.js
import express from 'express';
import { createServer } from 'http';
import { Server } from 'socket.io';
import path from 'path';
import { fileURLToPath } from 'url';
import { spawn } from 'child_process';

const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);

const app = express();
const httpServer = createServer(app);
const io = new Server(httpServer, {
  cors: { origin: "*" }
});

const PORT = process.env.WEB_PORT || 3000;

// Servir arquivos estáticos
app.use(express.static(path.join(__dirname, '../../public')));

app.get('/', (req, res) => {
  res.sendFile(path.join(__dirname, '../../public/index.html'));
});

let robotProcess = null;

io.on('connection', (socket) => {
  console.log('🔌 Cliente conectado');

  socket.on('start-robot', (data) => {
    if (robotProcess) {
      socket.emit('log', { level: 'warn', msg: '⚠️ Robô já está rodando.' });
      return;
    }

    const mode = data.mode; // 'DEV' ou 'PROD'
    socket.emit('log', { level: 'info', msg: `🚀 Iniciando Robô no modo: ${mode}` });

    // Define as variáveis de ambiente para o processo filho
    const env = { ...process.env };
    
    if (mode === 'DEV') {
      env.MODO_TESTE = 'true';
      env.LOGIN_URL = 'http://localhost:3456/login';
      env.AGENDAMENTO_URL = 'http://localhost:3456/agendamento/atendimento.php';
      env.WFM_LOGIN_URL = 'http://localhost:3456/login';
      env.WFM_URL = 'http://localhost:3456/agendamento/atendimento.php';
      env.GPS_URL = 'http://localhost:3456/login';
      socket.emit('log', { level: 'info', msg: '🔧 Configurado para Mock Local (Porta 3456)' });
    } else {
      env.MODO_TESTE = 'false';
      // As URLs reais virão do .env original do sistema
      socket.emit('log', { level: 'info', msg: '🔧 Configurado para Produção (URLs do .env)' });
    }

    // Inicia o processo filho (o robô principal)
    // Usamos 'node index.js' na raiz do projeto
    const projectRoot = path.join(__dirname, '../..');
    robotProcess = spawn('node', ['index.js'], {
      cwd: projectRoot,
      env: env,
      stdio: ['ignore', 'pipe', 'pipe'] // Captura stdout e stderr
    });

    socket.emit('status', { running: true });

    // Captura a saída do robô e envia via socket
    robotProcess.stdout.on('data', (data) => {
      const text = data.toString();
      // Envia linha por linha ou bloco
      socket.emit('log', { level: 'info', msg: text.trim() });
    });

    robotProcess.stderr.on('data', (data) => {
      const text = data.toString();
      socket.emit('log', { level: 'error', msg: text.trim() });
    });

    robotProcess.on('close', (code) => {
      socket.emit('log', { level: code === 0 ? 'success' : 'error', msg: `🏁 Robô finalizado (Código: ${code})` });
      socket.emit('status', { running: false });
      robotProcess = null;
    });

    robotProcess.on('error', (err) => {
      socket.emit('log', { level: 'error', msg: `❌ Erro ao iniciar processo: ${err.message}` });
      socket.emit('status', { running: false });
      robotProcess = null;
    });
  });

  socket.on('disconnect', () => {
    console.log('🔌 Cliente desconectado');
    // Opcional: Matar o robô se o cliente desconectar?
    // if (robotProcess) robotProcess.kill();
  });
});

httpServer.listen(PORT, () => {
  console.log(`
╔════════════════════════════════════════════════════════╗
║  🌐 Dashboard do Auto Robô v2.0                        ║
║  Acesse: http://localhost:${PORT}                       ║
╚════════════════════════════════════════════════════════╝
  `);
});
```

### 2. O Problema Provável: Falta de Logs no `stdout`

O `spawn` captura apenas o que vai para `console.log` (stdout) e `console.error` (stderr). Se o seu logger estiver configurado para escrever apenas em arquivos ou se houver um erro antes do `console.log`, o Dashboard não vê nada.

**Solução Rápida:** Adicione logs explícitos no início da execução do robô.

No seu arquivo principal de lógica (ex: `src/automation-runner.js` ou `index.js`):

```javascript
export const runAutomation = async () => {
  console.log('🚀 [ROBO] Iniciando função runAutomation...'); // Log crítico para debug
  
  let browser = null;
  try {
    console.log('📦 [ROBO] Chamando initBrowser...');
    const { browser: br, page } = await initBrowser();
    browser = br;
    
    console.log('✅ [ROBO] Navegador inicializado. URL atual:', page.url());
    
    // ... resto do código ...
    
  } catch (error) {
    console.error('❌ [ROBO] Erro na automação:', error.message);
    throw error;
  } finally {
    if (browser) {
      console.log('🔒 [ROBO] Fechando navegador...');
      await browser.close();
    }
    console.log('🏁 [ROBO] Processo finalizado.');
  }
};
```

```js
// index.js - Entry Point do Auto Robô v2.0
import { initBrowser, closeBrowser } from './src/browser/index.js';
import { loginService, injectionService, navigationService } from './src/index.js';
import { logger, automationConfig, systemsConfig } from './src/index.js';
import { captureAndSave } from './src/utils/screenshots.js';
import { sleep } from './src/utils/wait.js';

console.log('🤖 [ROBÔ] Processo iniciado via Dashboard...');

const runAutomation = async () => {
  let browser = null;
  let page = null;

  try {
    logger.info('main', '🚀 Iniciando automação...');
    
    // 1. Inicializa navegador
    const { browser: br, page: pg } = await initBrowser();
    browser = br;
    page = pg;

    // 2. Login no GOI
    logger.info('main', '🔐 Etapa 1: Login no GOI');
    await loginService.loginGOI(page);
    await sleep(1000);

    // 3. Login no WFM (se habilitado)
    if (systemsConfig.abrirWFM) {
      logger.info('main', '🔐 Etapa 2: Login no WFM');
      await loginService.loginWFM(page);
      await sleep(1000);
    }

    // 4. Abre abas adicionais
    logger.info('main', '🗂️ Etapa 3: Abrindo abas adicionais');
    await navigationService.openSystemTabs(browser, page);

    // 5. Executa Injeção D0
    logger.info('main', '💉 Etapa 4: Executando Injeção D0');
    await injectionService.executarInjecaoD0(page);

    // 6. Busca e atribui atividade (se configurado)
    if (automationConfig.atividadeTeste) {
      logger.info('main', '🎯 Etapa 5: Buscando atividade de teste');
      await injectionService.buscarEAtribuirAtividade(page, automationConfig.atividadeTeste);
    }

    // 7. Screenshot final
    await captureAndSave(page, 'conclusao');
    logger.success('main', '✅ Automação concluída com sucesso!');

  } catch (error) {
    logger.error('main', `❌ Erro crítico: ${error.message}`, error);
    if (page) await captureAndSave(page, 'erro_critico');
    process.exitCode = 1;
  } finally {
    if (browser) {
      logger.info('main', '🔒 Finalizando navegador...');
      await closeBrowser(browser);
    }
    logger.info('main', '🔚 Processo do robô encerrado.');
  }
};

// 🟢 EXECUTA SEMPRE AO SER CHAMADO (sem condições frágeis)
runAutomation().catch(err => {
  console.error('💥 Erro fatal não tratado:', err);
  process.exit(1);
});
```

io.on('connection', (socket) => {
  console.log('🔌 Cliente conectado ao Dashboard');

  socket.on('start-robot', (data) => {
    if (robotProcess) {
      socket.emit('log', { level: 'warn', msg: '⚠️ Robô já está rodando. Aguarde finalizar.' });
      return;
    }

    const mode = data.mode; // 'DEV' ou 'PROD'
    socket.emit('log', { level: 'info', msg: `🚀 Iniciando Robô no modo: ${mode}` });

    // Define as variáveis de ambiente para o processo filho
    // Copia o env atual mas permite sobrescrita segura
    const env = { ...process.env };
    
    if (mode === 'DEV') {
      env.MODO_TESTE = 'true';
      env.LOGIN_URL = 'http://localhost:3456/login';
      env.AGENDAMENTO_URL = 'http://localhost:3456/agendamento/atendimento.php';
      env.WFM_LOGIN_URL = 'http://localhost:3456/login';
      env.WFM_URL = 'http://localhost:3456/agendamento/atendimento.php';
      env.GPS_URL = 'http://localhost:3456/login';
      
      // Força HEADLESS false para ver o que acontece no modo DEV
      env.HEADLESS = 'false'; 
      
      socket.emit('log', { level: 'info', msg: '🔧 Configurado para Mock Local (Porta 3456)' });
      socket.emit('log', { level: 'warn', msg: '⚠️ Certifique-se de ter rodado "npm run mock" em outro terminal!' });
    } else {
      env.MODO_TESTE = 'false';
      // Mantém as URLs do .env original
      socket.emit('log', { level: 'info', msg: '🔧 Configurado para Produção (URLs do .env)' });
    }

    // Caminho absoluto para a raiz do projeto
    const projectRoot = path.join(__dirname, '../..');
    
    // Verifica se index.js existe antes de rodar
    const indexPath = path.join(projectRoot, 'index.js');
    if (!fs.existsSync(indexPath)) {
        socket.emit('log', { level: 'error', msg: `❌ Arquivo não encontrado: ${indexPath}` });
        return;
    }

    console.log(`[SERVER] Iniciando processo filho: node index.js em ${projectRoot}`);

    // Inicia o processo filho
    robotProcess = spawn('node', ['index.js'], {
      cwd: projectRoot,
      env: env,
      stdio: ['ignore', 'pipe', 'pipe'] // Captura stdout e stderr
    });

    socket.emit('status', { running: true });

    // Buffer para acumular dados parciais (caso venham quebrados)
    let stdoutBuffer = '';
    let stderrBuffer = '';

    // Captura STDOUT (console.log do robô)
    robotProcess.stdout.on('data', (data) => {
      const text = data.toString();
      stdoutBuffer += text;
      
      // Processa linha por linha para ficar bonito no dashboard
      const lines = stdoutBuffer.split('\n');
      // O último elemento pode estar incompleto, guarda no buffer
      stdoutBuffer = lines.pop(); 

      lines.forEach(line => {
        if (line.trim()) {
            // Tenta detectar nível de log pelo conteúdo, senão usa info
            let level = 'info';
            if (line.includes('❌') || line.includes('ERROR')) level = 'error';
            if (line.includes('⚠️') || line.includes('WARN')) level = 'warn';
            if (line.includes('✅') || line.includes('SUCCESS')) level = 'success';
            
            socket.emit('log', { level, msg: line });
        }
      });
    });

    // Captura STDERR (Erros de sistema/node)
    robotProcess.stderr.on('data', (data) => {
      const text = data.toString();
      stderrBuffer += text;
      const lines = stderrBuffer.split('\n');
      stderrBuffer = lines.pop();

      lines.forEach(line => {
        if (line.trim()) {
            socket.emit('log', { level: 'error', msg: line });
        }
      });
    });

    robotProcess.on('close', (code) => {
      console.log(`[SERVER] Processo filho encerrou com código ${code}`);
      const finalMsg = code === 0 ? '🏁 Robô finalizado com sucesso!' : `💥 Robô encerrado com erro (Código: ${code})`;
      socket.emit('log', { level: code === 0 ? 'success' : 'error', msg: finalMsg });
      socket.emit('status', { running: false });
      robotProcess = null;
    });

    robotProcess.on('error', (err) => {
      console.error('[SERVER] Erro ao spawnar processo:', err);
      socket.emit('log', { level: 'error', msg: `❌ Erro crítico ao iniciar: ${err.message}` });
      socket.emit('status', { running: false });
      robotProcess = null;
    });
  });

  socket.on('disconnect', () => {
    console.log('🔌 Cliente desconectado');
    // Opcional: Se quiser que o robô pare quando fechar a aba do navegador
    // if (robotProcess) {
    //   robotProcess.kill();
    //   robotProcess = null;
    // }
  });
});

### 1. Pilar Técnico: Resiliência e "Auto-Cura"
O maior inimigo da automação web é a instabilidade da rede ou lentidão do sistema alvo. Para um autista, erros inesperados geram ansiedade. Vamos blindar o código.

*   **Implementar "Retry Pattern" Inteligente:**
    *   *Problema:* Se o elemento não carregar na primeira tentativa, o robô quebra.
    *   *Solução:* Crie uma função utilitária `executarComTentativa(fn, tentativas = 3)` em `src/utils/retry.js`.
    *   *Benefício:* O robô tenta 3 vezes antes de falhar. Isso reduz falsos positivos e te dá paz de espírito.
*   **Health Check Contínuo no Dashboard:**
    *   *Melhoria:* No `public/index.html`, adicione um indicador visual (bolinha verde/vermelha) que pinga o Mock Server a cada 10 segundos.
    *   *Por que:* Você saberá *antes* de clicar em "Iniciar" se o ambiente está pronto, evitando o erro `ECONNREFUSED` surpresa.

### 2. Pilar Cognitivo (Aphantasia & Processamento): Feedback Não-Visual
Como você não cria imagens mentais e processa bem sons/texto estruturado, o feedback do robô deve ser **explícito**.

*   **Logs Estruturados em JSON (Opcional Avançado):**
    *   Além dos logs coloridos no terminal, gere um arquivo `execucao_log.json` ao final.
    *   Estrutura: `{ "etapa": "Login", "status": "SUCESSO", "tempo_ms": 1200, "timestamp": "..." }`.
    *   *Vantagem:* Você pode ler esse arquivo como texto puro para auditar o que aconteceu, sem precisar "imaginar" o fluxo.
*   **Feedback Sonoro (Beep Simples):**
    *   Adicione uma biblioteca simples como `node-beep` ou apenas um `console.log('\x07')` (bell character) quando o robô terminar ou der erro crítico.
    *   *Vantagem:* Você não precisa ficar olhando para a tela o tempo todo. O som avisa: "Acabei" ou "Deu erro".

### 3. Pilar Acadêmico (ADS): Documentação e Métricas
Para tirar nota máxima e impressionar recrutadores, o projeto precisa mostrar que você entende de **Engenharia de Software**, não apenas de "fazer funcionar".
